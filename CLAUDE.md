See [AGENTS.md](./AGENTS.md) for development, pull request, and package publishing
instructions. It is the single source of truth for how to work in this repo.

Quick reference:

- Run `/check` (or `npm run typecheck`, `npm test`, `npm run build`) before committing.
- Branch off `main`; never commit directly to `main`.
- The npm package name is `@ruddercode/rudder-core`; use that scope in install,
  publish, and release-verification instructions.
- Bump the version in the same PR as a user-facing change
  (`npm version patch --no-git-tag-version`), unless intentionally republishing
  the existing version under a renamed package.
- After opening a PR, use `/address-pr-comments` to triage review feedback.
- **Publishing is automatic on merge to `main`.** `.github/workflows/publish.yml`
  runs on every push to `main`: it publishes missing package name/version pairs
  to npmjs.org via OIDC and to GitHub Packages via `GITHUB_TOKEN`, pushes the
  `v<version>` tag after package registries are satisfied, and creates a GitHub
  Release with autogenerated notes. Just land the version bump or intentional
  package rename — no manual tagging. (Forgetting to bump = no new npmjs.org
  publish if the current name/version already exists.) Tests run separately in
  `.github/workflows/test.yml`.
- **PRs that publish artifacts get a release alert.**
  `.github/workflows/release-alert.yml` posts a sticky PR comment when merging
  will publish package or release artifacts (mirrors `publish.yml`'s checks), so
  a version bump or package rename never lands unnoticed.
- Path-resolving code must work in both the `.ts` dev tree and the compiled
  `.js` under `dist/` — see the `rudderArgv()` note in AGENTS.md.
- The dashboard (`rudder start`) and the digest read the **same** per-prompt tags
  (`prompt_tags` / `statsForDay`), so their numbers always agree. Tagging is
  out-of-band (never in the capture hook) and uses the shared rubric in
  `classify.ts`. Bump `TAGGER_VERSION` to re-tag history. See the "Stats pipeline"
  section in AGENTS.md.

## Slash commands

- `/check` (`.claude/commands/check.md`) — run the package checks and enforce
  Claude/Codex sync; mirrors the `check-changed-folders` Codex skill.
- `/address-pr-comments` (`.claude/commands/address-pr-comments.md`) — fetch and
  remediate open PR review comments; mirrors the `address-pr-comments` Codex skill.

## Gitmoji

Use [gitmoji](https://gitmoji.dev/) in all commit messages and PR titles, picking
the most specific emoji. Note `:zap:` (new features) and `:sparkles:` (performance)
are swapped from the standard guide. See the table in [AGENTS.md](./AGENTS.md#gitmoji).

## Claude / Codex parity

Any change to a Claude-side file (`CLAUDE.md`, `.claude/`) must be mirrored on the
Codex side (`AGENTS.md`, `.codex/`) in the same PR, and vice versa:

- `CLAUDE.md` <-> `AGENTS.md`
- `.claude/commands/check.md` <-> `.codex/skills/check-changed-folders/SKILL.md`
- `.claude/commands/address-pr-comments.md` <-> `.codex/skills/address-pr-comments/SKILL.md`
